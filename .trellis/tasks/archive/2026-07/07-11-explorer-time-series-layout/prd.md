# 优化 Cost Explorer 时间序列展示

## Goal

压缩 Cost Explorer 时间序列区域的默认页面高度，在不丢失查询结果、离线查看能力和精确数值明细的前提下，让用户先理解趋势，再按需检查完整数据。

## User Value

- 避免时间点数量直接决定整个页面高度，减少长距离滚动和左右栏高度失衡。
- 将趋势判断与精确明细分层，用户不必先阅读几十行表格才能看出时间变化。
- 保留 snapshot/export 场景中的完整查询结果，离线时仍可查看明细。

## Confirmed Facts

- Explorer 结果区当前采用左右两栏：左侧维度排行，右侧时间序列表格。
- 时间序列行数按“时间桶 x 分组维度”增长；当前前端最多渲染 80 行，但没有显式截断提示。
- 默认查询可以产生约 60 个 series points，已足以让右栏显著高于左栏并留下大量空白。
- 移动端会将两栏改为单列，但完整时间序列表仍会继续拉长页面。
- 项目现有“用量趋势”区域已采用“紧凑趋势图 + 默认折叠完整明细”的信息层级，可作为一致性参考。
- Explorer payload 已包含完成本次展示优化所需的聚合 rows 和 series；不需要浏览器透视原始 usage events。

## Requirements

- 时间序列的默认可见区域必须保持有界，不能随 series points 数量无限增长。
- 默认视图必须提供可用于判断时间变化的趋势摘要，而不仅是隐藏或截断原始表格。
- MVP 必须包含紧凑趋势图，并将完整时间序列表降级为默认折叠的明细视图。
- 趋势图最多展示总排行前 5 个维度；`Other` 若进入展示集合则占用一个名额，其余维度仍保留在完整明细中。
- 趋势图必须明确说明“前 5 个维度”的展示范围，不能让用户误以为未绘制的维度没有数据。
- 前 5 个维度使用共享时间轴的紧凑小多图呈现，每个维度独占一行，不使用多条折线重叠或依赖多色图例区分。
- 每个小图使用独立纵轴尺度突出自身变化形态，并显示峰值；界面必须明确标注“各维度独立刻度”，绝对量级比较继续由维度排行承担。
- 小多图覆盖响应中完整的时间桶集合；某个已出现时间桶缺少某维度数据时按零值绘制，不连接成未知的非零值。
- 完整明细必须可由用户在当前页面内按需展开，并保留时间、维度和当前指标值。
- 明细区域必须显示总数据点数量和查询时间范围；发生展示上限截断时必须明确提示。
- 明细继续保留 80 行展示上限；超过上限时展示最近 80 条并显示“已显示 80 / 共 N 条”，趋势图不受该表格上限影响。
- 展开的长表格必须使用内部限高滚动和固定表头，避免再次拉长整页。
- 桌面布局应减少排行栏下方的无效留白；移动端必须按清晰阅读顺序堆叠且不产生水平溢出。
- live dashboard 与 snapshot/export 必须复用相同的展示行为。
- 中英文文案、浅色/深色主题、键盘焦点和无动画偏好必须保持可用。
- 保持现有 Explorer API、查询参数、后端聚合语义和返回 payload 兼容。

## Acceptance Criteria

- [x] 使用 60 个或更多 series points 时，时间序列默认区域高度保持在一个桌面视口内，不再产生截图中的超长右栏。
- [x] 用户无需展开完整表格即可识别当前区间的主要时间变化。
- [x] 趋势区最多出现 5 个按总量选出的维度小图，每行标明维度、独立刻度和峰值，且图表范围说明未绘制维度仍可在明细查看。
- [x] 用户可以展开和收起明细；80 条以内展示全部，超过 80 条时展示最近 80 条时间点、维度和指标值并明确提示截断。
- [x] 明细入口展示 series point 总数与起止时间；任何前端展示上限都有可见说明。
- [x] 展开明细后，表头在内部纵向滚动期间保持可见。
- [x] 宽屏下排行与趋势区域视觉平衡；窄屏下按单列排列且没有文本、控件或表格溢出。
- [x] 查询筛选、运行分析、重置、live 加载和 snapshot/export 行为没有回归。
- [x] 中英文与浅色/深色主题均通过桌面和移动端截图检查。
- [x] 相关 Web 资产契约测试和仓库要求的最小质量门禁通过。

## Out of Scope

- 修改 Explorer SQL、Rust 查询模型、API schema 或查询参数。
- 引入第三方图表库。
- 为最多数十到低百条的明细实现服务端分页、无限滚动或虚拟列表。
- 重做 Cost Explorer 筛选控件、摘要指标或维度排行算法。
- 改变 Top N、Other、时间粒度或过滤器的业务语义。
